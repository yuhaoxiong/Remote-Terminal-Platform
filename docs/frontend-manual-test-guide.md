# 前端手工测试手册

## 1. 目的

本文档用于通过浏览器前端界面验收当前已完成的功能，覆盖：

- 登录、退出、修改密码、用户管理和角色权限
- 仪表盘、指标展示和异常设备摘要
- 设备管理、frps 导入、同步配置和文件管理
- 分组管理
- 远程 SSH 和 VNC
- 批量更新任务、目标预览、命令模板、执行结果和导出
- 定时任务、调度状态、执行记录和日志
- 告警中心、告警确认/恢复和规则配置
- 用户管理、启停账号和重置密码
- 操作日志筛选和 CSV 导出
- 系统诊断、安全提醒、迁移和监控可用性

本文档不替代后端自动化测试和 Postman 接口验收。它只描述当前前端
界面能够直接操作的测试路径。

## 2. 当前前端测试范围

当前业务前端实际集中在单页控制台中，左侧栏目为：

1. 仪表盘
2. 设备管理
3. 远程连接
4. 文件管理
5. 批量更新
6. 定时任务
7. 告警中心
8. 系统诊断
9. 操作日志
10. 用户管理（仅管理员）
11. 分组管理

### 2.1 可仅靠平台前端完成的测试

- 登录、退出、修改密码
- 仪表盘在已有数据下的列表、统计、图表和异常摘要展示
- 设备新增、编辑、删除、搜索、分组筛选、刷新状态
- 分组新增、编辑、删除、按分组跳转设备列表
- 批量更新演练任务创建、目标筛选、目标预览、执行、取消、导出
- 命令模板新增、编辑、套用、删除
- 定时任务演练模式新增、编辑、停用、启用、立即执行、查看执行记录和日志
- 告警中心查看摘要、筛选告警、确认/恢复告警、启停和保存规则
- 管理员创建用户、调整角色、启停账号和重置密码
- 运维账号登录后确认无"用户管理"入口
- 操作日志筛选和导出
- 系统诊断查看和刷新

### 2.3 Wave 22 企业后台界面验收重点

Wave 22 后前端采用“左侧导航 + 顶部状态栏 + 主内容区”的企业级运维后台布局。手工验收时除功能正确外，还需要检查：

- 顶栏展示平台模块标题、后端健康状态、调度器状态、当前用户、角色、修改密码和退出登录。
- 左侧导航按模块分组，当前页面高亮明确；运维账号不显示“用户管理”。
- 设备状态、告警等级、任务状态、通知投递状态使用统一中文状态标签。
- 表格无数据时显示中文空状态，不显示空白表格或 `undefined`。
- 设备缺少 SSH/VNC 端口或凭据时，连接按钮禁用并显示中文原因。
- 高风险操作均出现确认弹窗，包括删除设备、删除分组、删除文件、真实 SSH 执行、真实 SSH 定时任务、停用用户、重置密码、删除通知通道和删除通知策略。
- `403` 权限不足只显示中文无权限提示，不返回登录页；`401` 会话失效才返回登录页。
- 远程连接、文件管理、告警通知和批量任务失败只在对应区域提示，不影响其他模块继续操作。

### 2.2 依赖外部条件的前端测试

| 功能 | 额外条件 |
| --- | --- |
| frps 导入 | 后端能访问 frps Dashboard，且有可识别 TCP 代理 |
| 设备同步配置 | 设备记录已存在；仅验证前端生成和展示 frpc 配置文本 |
| 文件管理本地模式 | 后端 `FILE_BACKEND=local` 即可 |
| 文件管理真实设备模式 | 后端 `FILE_BACKEND=sftp`，设备 SSH 端口和凭据可用 |
| 远程 SSH | 设备 SSH 端口、凭据、frp 链路和 `/api/ws/` 可用 |
| 远程 VNC | 设备 VNC 端口、VNC 服务、frp 链路和 `/api/ws/` 可用 |
| 真实 SSH 批量更新 | 至少一台可安全操作的测试设备 |
| 真实 SSH 定时任务 | 同上，且要明确命令影响范围 |

## 3. 测试环境准备

### 3.1 环境要求

测试前确认：

- 前端能正常打开。
- 后端健康检查正常。
- 前端和后端代理链路正常。
- WebSocket 代理已可用，正式环境推荐同域代理 `/api` 和 `/api/ws`。
- 浏览器允许下载文件，便于验证 CSV 和设备文件下载。

推荐先访问：

```text
GET /api/health
```

健康结果应至少满足：

- HTTP `200`
- `status=ok`
- `database=ok`

### 3.2 测试账号

使用部署环境管理员账号登录。

首次生产类测试前应确认：

- `JWT_SECRET_KEY` 已配置。
- 管理员密码不是默认值。
- 若要保存真实设备凭据，`CREDENTIAL_ENCRYPTION_KEY` 已配置。

### 3.3 测试数据建议

准备以下数据，避免把所有场景都压在一台设备上：

| 数据 | 用途 |
| --- | --- |
| 分组 `UI-测试组-A` | 分组新增、编辑、设备归组 |
| 分组 `UI-测试组-B` | 分组筛选和迁移 |
| 设备 `ui-edge-dryrun-01` | 无真实远程依赖的设备管理和演练任务 |
| 设备 `ui-edge-remote-01` | 已连通 frp 的 SSH/VNC/SFTP 测试设备 |
| 标签 `ui-test` | 设备搜索和批量目标筛选 |
| 项目号 `ui-project-a` | 设备筛选、批量更新、定时任务目标 |
| 小文件 `ui-upload.txt` | 文件上传、下载、删除 |

真实设备测试建议使用只读命令：

```text
hostname
whoami
uptime
```

不要在首轮前端验收中执行升级、重启、删除文件、改配置等有副作用命令。

每个用例至少记录：

| 字段 | 说明 |
| --- | --- |

## 5. 快速冒烟路径

先跑下面这条路径。如果这里失败，不建议继续做全量手工测试。

1. 打开前端登录页。
2. 使用管理员账号登录。
3. 切换到“仪表盘”，确认页面有统计卡片和数据区域。
4. 切换到“设备管理”，确认设备表能加载。
5. 新建一个演练用设备。
6. 切换到“分组管理”，新建一个测试分组。
7. 编辑设备并归入该分组。
8. 切换到“批量更新”，创建 `dry_run` 的 `hostname` 任务。
9. 预览目标，保存任务，执行任务。
10. 切换到“操作日志”，确认可看到本轮操作记录。
11. 切换到“用户管理”，创建一个运维测试账号。
12. 退出后使用运维账号登录，确认左侧不显示“用户管理”。
13. 切换到“系统诊断”，刷新并确认诊断数据能加载。

## 6. 登录与账号测试

### FE-AUTH-01 管理员登录成功

步骤：

1. 打开登录页。
2. 输入管理员用户名。
3. 输入正确密码。
4. 点击“登录”。

预期：

- 进入控制台首页。
- 左侧功能栏目可见。
- 管理员可看到“用户管理”入口。
- 不显示登录错误。

### FE-AUTH-02 登录失败

步骤：

1. 在登录页输入错误密码。
2. 点击“登录”。

预期：

- 停留在登录页。
- 显示登录失败提示，不重复显示同一条提示。
- 不进入业务页面。

### FE-AUTH-03 退出登录

步骤：

1. 已登录后点击右上角退出按钮。

预期：

- 返回登录页。
- 重新刷新页面后仍需要登录。

### FE-AUTH-04 修改当前账号密码

步骤：

1. 已登录后点击顶栏“修改密码”。
2. 测试新密码为空、长度不足、两次输入不一致。
3. 输入正确旧密码和满足要求的新密码。
4. 点击“保存并重新登录”。
5. 使用新密码重新登录。

预期：

- 前三类非法输入被前端拦截并显示提示。
- 正确修改后退出登录。
- 旧密码失效，新密码可登录。

备注：

- 若此环境还要继续测试，提前记录新密码。

## 7. 用户与权限测试

### FE-USER-01 创建运维账号

步骤：

1. 使用管理员账号登录。
2. 进入“用户管理”。
3. 点击“新建用户”。
4. 输入用户名、至少 8 位初始密码，角色选择“运维人员”。
5. 点击“创建用户”。

预期：

- 用户列表新增该账号。
- 角色显示为“运维人员”。
- 操作日志中出现 `user.create`。

### FE-USER-02 运维账号入口限制

步骤：

1. 退出管理员账号。
2. 使用刚创建的运维账号登录。

预期：

- 能进入控制台。
- 顶栏显示当前用户名和“运维人员”。
- 左侧不显示“用户管理”入口。

### FE-USER-03 运维账号无高风险权限

步骤：

1. 使用运维账号登录。
2. 创建或编辑一台测试设备。
3. 创建并执行 `dry_run` 批量任务。
4. 尝试通过接口或页面触发删除设备、真实 SSH 批量任务或告警通知配置。

预期：

- 设备创建或编辑成功。
- `dry_run` 批量任务成功。
- 高风险操作返回无权限提示，页面不退出登录。
- 操作日志中出现 `auth.forbidden`。

### FE-USER-04 启停和重置密码

步骤：

1. 使用管理员账号登录。
2. 进入“用户管理”。
3. 对运维测试账号执行“重置密码”。
4. 停用该账号。
5. 尝试使用该账号登录。
6. 重新启用该账号并使用新密码登录。

预期：

- 重置密码成功后旧密码失效。
- 停用账号不能登录。
- 重新启用后可用新密码登录。
- 后端不允许停用或降级最后一个启用管理员。

## 8. 仪表盘测试

### FE-DASH-01 总览加载

步骤：

1. 登录后进入“仪表盘”。

预期：

- 设备统计、在线统计、风险统计和已完成任务统计能显示。
- 设备状态分布和资源风险图表区域可见。
- 页面无持续加载错误。

### FE-DASH-02 资源快照

步骤：

1. 准备一台已有最新指标的设备。
2. 回到“仪表盘”刷新页面。
3. 查看“资源快照”。

预期：

- 有指标设备显示 CPU、内存、磁盘值。
- 无指标设备显示“暂无指标”，不能误显示为 `0%`。
- 指标读取失败时出现明确提示，不应直接退出登录。

### FE-DASH-03 异常设备摘要

步骤：

1. 准备离线、未知或高指标设备数据。
2. 查看“异常设备”区域。

预期：

- 离线、未知、指标过期、高 CPU、高内存、高磁盘等摘要按数据出现。
- 无异常时显示空状态。

## 9. 设备管理测试

### FE-DEV-01 设备列表、搜索和分组筛选

步骤：

1. 进入“设备管理”。
2. 在搜索框分别输入设备名称、序列号、项目号、标签。
3. 使用分组筛选。
4. 清除筛选。

预期：

- 表格按搜索和分组条件收敛。
- 清除后恢复列表。

### FE-DEV-02 创建设备

步骤：

1. 点击“新建设备”。
2. 输入设备名称、设备序列号、项目号、位置、标签。
3. 选择分组。
4. 配置 SSH 用户、凭据类型和测试密码。
5. 保存。

预期：

- 设备出现在表格中。
- 分组、标签、项目号显示正确。
- 不应在表格中回显 SSH 明文密码。

建议数据：

```text
名称: ui-edge-dryrun-01
序列号: UI-DRYRUN-001
项目号: ui-project-a
标签: ui-test,dry-run
```

### FE-DEV-03 编辑设备

步骤：

1. 点击某设备“编辑”。
2. 修改名称、位置、标签或分组。
3. 不填 SSH 密码并保存。
4. 再次编辑设备。

预期：

- 非凭据字段更新成功。
- 编辑页不会回填旧 SSH 密码。
- 密码留空表示不修改已有凭据。

### FE-DEV-04 刷新设备状态

步骤：

1. 在设备表点击“刷新状态”。

预期：

- 状态列刷新。
- 若刷新失败，界面显示失败提示，不出现静默失败。

### FE-DEV-05 删除设备

步骤：

1. 选择临时测试设备。
2. 点击“删除”。
3. 先取消确认框。
4. 再次删除并确认。

预期：

- 取消后设备保留。
- 确认后设备从列表移除。
- 操作日志有对应记录。

### FE-DEV-06 同步配置展示

步骤：

1. 在设备表点击“同步配置”。
2. 查看生成的 frpc 配置文本。
3. 尝试复制配置。

预期：

- 弹出配置区域。
- 配置内容可见。
- 浏览器支持剪贴板时提示复制成功；不支持时给出手动复制提示。

### FE-DEV-07 frps 导入

前置：

- 后端能访问 frps Dashboard。
- 已准备 Dashboard 地址、账号密码和端口范围。

步骤：

1. 在“设备管理”点击“导入 frps”。
2. 输入 Dashboard 地址、用户名、密码、项目号、位置和端口范围。
3. 点击“开始导入”。
4. 对同一批数据再次导入。

预期：

- 首次导入显示发现、新增、同步、跳过、冲突统计。
- 明细表显示每台设备导入结果。
- 重复导入不应无脑重复创建设备。
- Dashboard 不可达或认证失败时显示中文错误，表单输入不应被清空。

## 12. 远程连接测试

### FE-REMOTE-01 不满足条件时的禁用提示

步骤：

1. 进入“远程连接”。
2. 选择没有 SSH 端口、VNC 端口或凭据的设备。

预期：

- SSH/VNC 操作区显示原因。
- 不满足条件的连接按钮不可用。

### FE-REMOTE-02 SSH 连接

前置：

- 测试设备 SSH frp 端口可达。
- 设备凭据可用。
- Nginx 或开发代理支持 WebSocket。

步骤：

1. 进入“远程连接”。
2. 搜索并选择 `ui-edge-remote-01`。
3. 点击“连接 SSH”。
4. 在 xterm 终端输入 `hostname`。
5. 输入 `whoami`。
6. 点击“断开 SSH”。

预期：

- 状态从连接中进入已连接。
- 终端可看到命令输出。
- 断开后状态更新。
- 连接失败时错误只显示在远程区域，不应无故退出登录。

### FE-REMOTE-03 VNC 连接

前置：

- 测试设备 VNC 服务和 frp VNC 端口可用。

步骤：

1. 选择测试设备。
2. 点击“连接 VNC”。
3. 确认画面区域出现远程画面。
4. 点击“全屏”。
5. 退出全屏后点击“断开 VNC”。

预期：

- VNC 状态可见。
- 画面可显示。

## 16. 告警中心测试

### FE-ALERT-01 告警摘要和列表

步骤：

1. 进入“告警中心”。
2. 点击“刷新”。
3. 使用状态、级别和来源筛选。

预期：

- 活跃告警、严重告警、未确认和最近告警时间可见。
- 告警列表展示标题、描述、级别、来源、状态、阈值和时间。
- 无告警时显示空状态，不应报错或退出登录。

### FE-ALERT-02 确认和恢复告警

步骤：

1. 找到一条未处理告警。
2. 点击“确认”。
3. 再点击“恢复”。

预期：

- 确认后调用后端并刷新列表，操作日志会记录 `alert.acknowledge`。
- 恢复后告警状态变为已恢复，操作日志会记录 `alert.resolve`。
- 页面不展示设备密码、Token 或其他敏感字段。

### FE-ALERT-03 告警规则

步骤：

1. 查看“告警规则”表。
2. 修改规则启用状态、级别或阈值。
3. 点击“保存”。

预期：

- 保存后规则列表刷新。
- CPU/内存/磁盘阈值和指标冻结窗口可编辑。
- 空规则或非法阈值应由后端返回校验错误，前端显示失败提示。

### FE-ALERT-04 外部通知

步骤：

1. 在“告警中心”查看“外部通知”区域。
2. 新增 Webhook 通道，填写名称、Webhook 地址、请求头 JSON 和超时秒数。
3. 点击“测试”，确认目标 Webhook 收到测试 payload。
4. 新增通知策略，选择通道、最低级别和触发事件。
5. 触发一条严重告警后查看“最近投递”。
6. 对失败投递点击“重试”。
7. 删除通道前先删除引用该通道的策略。

预期：

- 页面展示启用通道、启用策略、失败投递和待重试数量。
- Webhook 地址和请求头值不会在前端明文回显，只显示脱敏地址和请求头 key。
- 未配置 `CREDENTIAL_ENCRYPTION_KEY` 时，保存 Webhook 通道失败并出现中文提示。
- 通道仍被策略引用时，删除通道失败并提示先删除策略。
- 重试成功后投递状态刷新。

下面几项建议至少测试一次：

| ID | 异常路径 | 预期 |

| FE-ERR-09 | Webhook 通道缺少加密密钥或目标不可达 | 外部通知区域显示失败提示，不影响告警列表和规则表 |

## 19. 测试完成判定

满足以下条件可认为当前前端功能已完成一轮手工验收：

1. 第 5 节冒烟路径通过。
2. 登录、设备、分组、批量更新、定时任务、告警、日志、诊断主路径通过。
3. 文件管理按当前部署模式完成验证。
4. frps、SSH、VNC、SFTP、真实 SSH 批量执行按环境能力完成验证或明确标记阻塞。
5. 下载类动作已验证：
   - 操作日志 CSV
   - 批量更新结果 CSV
   - 设备文件下载
6. 未发现前端明文回显密码、Token、私钥或凭据加密密钥。
7. 每个失败或阻塞项都记录了浏览器现象、测试数据和外部依赖状态。

## 20. 响应式和视觉检查

建议至少检查以下尺寸：

| 尺寸 | 检查重点 |
| --- | --- |
| 1920x1080 | 顶栏状态、仪表盘指标卡、设备表格、远程终端区域是否有足够信息密度 |
| 1366x768 | 左侧导航、顶部状态栏、表格操作列、详情抽屉是否遮挡或换行异常 |
| 窄屏，约 768px 宽 | 导航、工具栏、筛选表单、远程连接 Tabs、抽屉宽度是否仍可操作 |

逐页检查：

- 登录页左右分栏在小屏下应能纵向排列，错误提示不重复。
- 仪表盘指标卡和中部面板不应横向溢出。
- 设备管理表格操作列可见，详情抽屉在小屏下接近全屏但不遮挡关闭按钮。
- 远程连接左侧设备列表和右侧 SSH/VNC 工作区在窄屏下可纵向阅读，终端区域不应高度为 0。
- 批量更新任务配置、目标预览和任务详情不应互相遮挡。
- 告警中心规则、通知配置和投递记录在窄屏下能滚动查看。
- 操作日志详情抽屉文本应自动换行，不展示横向滚动的敏感大文本。
- 系统诊断卡片应自动换行，风险提醒不与其他卡片重叠。

## 21. 推荐测试顺序

完整测试建议按下面顺序执行：

1. 账号与登录
2. 系统诊断
3. 分组管理
4. 设备管理
5. 仪表盘
6. 文件管理
7. 批量更新演练
8. 定时任务演练
9. 告警中心
10. 操作日志和导出
11. frps 导入
12. 远程 SSH
13. 远程 VNC
14. 告警外部通知
15. SFTP 和真实 SSH 任务

这样能先完成低风险 UI 主路径，再进入依赖 frp 和真实设备的联调场景。
