# Wave 22 需求文档：前端企业级运维后台重构与体验统一

> 阶段：`requirement_doc`
> 状态：待确认，执行前冻结
> 基线：Wave 21 已完成多用户、角色权限和会话审计基础。当前前端已经开始引入左侧导航、顶部状态栏、中文登录页、设备详情抽屉、远程连接工作区、文件管理入口、权限感知入口和若干通用组件，但主应用仍存在页面逻辑集中、部分页面视觉层级不一致、状态反馈不完整、响应式验收不足等问题。本轮目标是在不改写后端接口的前提下，把前端推进为现代化企业级运维管理后台。

## 1. 背景

截至 Wave 21，平台已经具备：

- 管理员和运维人员登录，JWT access/refresh token，修改密码，角色权限和 `401`/`403` 差异化处理。
- 设备 CRUD、分组、标签、frps Dashboard 导入、设备级 SSH 凭据、SSH/VNC 端口和最近指标。
- Web SSH、VNC、设备文件管理、设备详情、设备状态、指标采集和系统诊断。
- 批量更新任务：目标预览、dry-run、真实 SSH 执行、失败设备重试、结果 CSV 导出和 WebSocket 进度快照。
- 定时任务：配置管理、调度器状态、立即执行、执行记录和执行日志。
- 告警中心：活跃告警、告警规则、确认/恢复、自动恢复、通知通道、通知策略、投递记录和失败重试。
- 操作日志、用户管理、README、API 文档、部署文档、前端手工测试指南和 Postman Collection。

当前前端已经开始进行企业后台化改造：

- 新增 `LayoutShell.vue`、`AppSidebar.vue`、`AppTopbar.vue`、`StatusBadge.vue`、`MetricCard.vue`、`DeviceDetailDrawer.vue` 等组件。
- 登录页已改为左右分栏，并保留登录测试入口。
- 主界面已具备左侧导航、顶部状态栏、主内容区结构。
- 设备管理新增筛选、详情抽屉、SSH/VNC/文件入口和端口/凭据缺失禁用原因。
- 远程连接已改为左侧设备列表和右侧 SSH/VNC/日志 Tabs。
- 文件管理已成为一级导航入口。
- 批量更新、定时任务、告警中心已开始按 `admin`/`operator` 控制高风险入口。
- 前端测试已补充健康接口 mock、登录和部分权限可见性场景。

仍需继续优化的问题：

- `App.vue` 仍承载大量页面逻辑，后续维护和测试成本较高。
- 仪表盘、设备管理、远程连接、文件管理、批量更新、定时任务、告警中心、系统诊断、操作日志、用户管理之间的视觉密度和交互状态还不完全统一。
- 部分页面缺少标准化 Skeleton、Loading、Empty、Error、权限不足、禁用原因和危险操作确认。
- 远程连接、VNC、xterm、文件操作和批量任务详情这类强交互区域需要更明确的局部错误处理，避免误影响登录态。
- 响应式适配尚未按 1366x768、1920x1080 和窄屏做系统验收。
- 前端测试对企业后台关键路径和权限降级场景覆盖不足。

## 2. 目标

Wave 22 完成后，前端应达到以下状态：

1. 整体界面呈现现代化企业级运维管理后台风格，采用左侧导航、顶部状态栏、主内容区布局。
2. 页面视觉统一使用科技蓝主色、浅灰背景、白色内容容器、清晰的状态色和一致的表格/抽屉/弹窗样式。
3. 所有前端可见文案默认中文，后续新增前端文案也保持中文。
4. 保留现有后端 API、JWT 鉴权、refresh token、权限判断和测试逻辑，不重写后端接口。
5. 前端继续使用 Vue 3 Composition API、TypeScript 和 Element Plus 组件体系。
6. `operator` 角色隐藏或禁用用户管理、高风险真实 SSH 操作、告警规则编辑和通知配置入口。
7. `401` 继续回登录页，`403` 显示中文无权限提示并保留当前页面。
8. 远程连接失败只在远程连接区域显示错误，不误清理登录态，除非接口明确返回 `401` 或 `403`。
9. 设备管理、远程连接、批量更新、定时任务、告警中心、系统诊断、操作日志和用户管理页面具备统一的加载、空状态、错误和禁用原因表达。
10. 逐步拆分 `App.vue` 页面级逻辑，形成更易维护的视图组件和业务组件。
11. 前端测试覆盖登录、导航、角色可见性、权限不足、设备连接禁用、远程连接失败和危险操作确认等关键路径。
12. 本轮完成后提供可运行验证结果，至少包含前端测试、类型检查、构建和浏览器冒烟。

## 3. 非目标

本轮不实现以下内容：

- 不重写后端 API，不新增后端业务能力。
- 不改变 JWT、refresh token、后端权限矩阵和数据库结构。
- 不引入新的前端框架，不从 Vue/Element Plus 迁移到其他体系。
- 不重写 xterm/noVNC 的核心连接协议，只优化承载布局、状态提示和错误反馈。
- 不实现完整深色主题或主题编辑器。
- 不实现移动端完整运维体验；窄屏只要求可读、可操作、不遮挡。
- 不重做完整品牌系统或新增复杂插画资产。
- 不为了视觉改造删除现有测试或降低测试断言。
- 不把前端隐藏作为安全边界，真正权限仍以后端返回为准。

## 5. 页面需求

### 5.1 登录页

要求：

- 保持左右分栏布局。
- 左侧展示平台定位：设备接入、远程运维、批量更新、告警闭环。
- 右侧为登录卡片，包含用户名、密码、登录按钮和错误提示。
- 输入为空、密码错误和后端失败只显示一条中文错误。
- 登录成功进入仪表盘。
- 不在登录页展示后端敏感配置或调试信息。

验收：

- 空用户名或空密码只出现一条错误提示。
- 错误密码只出现一条错误提示。
- 登录成功后看不到登录按钮，默认显示仪表盘。

### 5.2 仪表盘

要求：

- 顶部指标卡片展示设备总数、在线设备、离线/未知设备、活跃告警、待执行任务。
- 中部展示设备状态分布、资源异常排行和最近指标状态。
- 底部展示最近告警、最近批量任务和最近操作日志。
- 异常设备和告警支持点击跳转到对应模块或详情。
- 图表可以继续使用轻量 CSS 展示，也可在后续确认后引入 ECharts。

验收：

- 指标为空时显示清晰空状态，不显示 undefined/null。
- 卡片在 1366x768 下不遮挡、不横向溢出。

### 5.3 设备管理

要求：

- 筛选支持关键词、分组、状态、标签、项目号。
- 表格列包含设备名称、状态、项目号、分组、部署位置、SSH 端口、VNC 端口、最近指标时间、操作。
- 操作包含详情、SSH、VNC、文件、同步配置、编辑、删除。
- 右侧抽屉展示设备详情、frpc 配置摘要、SSH 凭据状态、最近指标和快捷入口。
- 缺少 SSH/VNC 端口或凭据时，禁用对应按钮并显示中文原因。
- 删除、重启、真实连接等高风险操作必须有确认。

验收：

- 端口缺失时 SSH/VNC 按钮不可点击，tooltip 或提示说明原因。
- 打开详情抽屉不影响列表筛选状态。

### 5.4 文件管理

要求：

- 文件管理作为一级导航入口保留。
- 页面应先选择设备，再进入设备文件面板。
- 设备列表复用设备管理的状态标签和基础筛选体验。
- 文件上传、下载、删除失败时只在文件管理区域显示错误。
- 文件删除和覆盖类操作需要确认弹窗。

验收：

- 未选择设备时显示中文空状态。
- 选择设备后显示对应设备的文件管理面板。

### 5.5 远程连接

要求：

- 使用左侧设备列表和右侧远程工作区。
- 左侧支持按名称、序列号、项目号、位置搜索设备。
- 右侧显示当前设备状态、SSH/VNC 端口、连接、断开和全屏按钮。
- 使用 Tabs 展示 SSH 终端、VNC 桌面和连接日志。
- SSH 区域保持黑色终端风格，承载 xterm。
- VNC 区域承载 noVNC，提供连接、断开、全屏。
- 连接失败只在远程连接区域显示错误，不误清登录态。

验收：

- SSH 创建失败后仍停留在远程连接页面。
- VNC 创建失败后仍停留在远程连接页面。
- 没有选择设备时显示空状态。

### 5.6 批量更新

要求：

- 新建任务区域分为任务配置、目标筛选、目标设备预览。
- 执行模式区分 `dry_run` 和 `ssh_command`。
- `ssh_command` 模式仅 admin 可见或可提交。
- `ssh_command` 必须显示醒目风险提示和二次确认。
- 任务列表展示状态、进度、成功数量、失败数量、创建人和操作。
- 任务详情展示 WebSocket 实时进度、单设备结果、stdout/stderr 摘要、失败原因、失败设备重试入口和 CSV 导出。

验收：

- operator 看不到或不能提交真实 SSH 执行。
- admin 提交真实 SSH 执行前必须确认。
- 执行失败设备可以进入失败处理入口。

### 5.7 定时任务

要求：

- 展示调度器状态、启用任务数、失败执行数、最近扫描时间。
- 支持创建、编辑、启停、立即执行、删除。
- operator 只能查看和执行允许范围内的任务，不能管理高风险配置。
- 任务列表展示 cron/interval 表达式、下次执行时间、最近执行结果。
- 执行记录和执行日志通过抽屉或详情区域展示。

验收：

- 非 admin 无法创建、编辑、删除、启停任务。
- 真实 SSH 定时任务执行前有确认或权限阻断。

### 5.8 告警中心

要求：

- 顶部展示活跃告警、严重告警、未确认告警、通知失败数量。
- 告警表格支持等级、状态、类型、设备、时间筛选。
- 支持确认、恢复操作。
- 告警规则支持阈值编辑和启停，但 operator 只读。
- 外部通知区域展示 Webhook 通道、通知策略、投递记录和失败重试，operator 不显示配置入口。
- 未配置凭据加密密钥时显示安全风险提示，不显示任何敏感明文。

验收：

- operator 可以确认/恢复告警，但不能编辑规则或通知配置。
- 通知配置错误不影响告警列表展示。

### 5.9 系统诊断

要求：

- 展示 API 健康、数据库状态、迁移状态、调度器状态、认证配置、SSH 主机密钥策略、文件后端、告警摘要、通知摘要和用户摘要。
- 安全风险使用 Alert 组件突出显示。
- 不展示密码、Token、私钥、Webhook 明文密钥或完整敏感 URL。

验收：

- 诊断页无敏感字段明文。
- 风险项为中文且能指导下一步处理。

### 5.10 操作日志

要求：

- 支持 action、target_type、status、用户、时间范围筛选。
- 表格展示操作者、动作、目标类型、状态、IP、时间。
- 支持分页和 CSV 导出。
- 操作详情通过抽屉或详情面板展示，请求/响应/执行日志需要脱敏。

验收：

- 空数据时显示“暂无操作日志”或同等中文说明。
- CSV 导出失败显示中文错误，不影响页面状态。

### 5.11 用户管理

要求：

- 仅 admin 可见。
- 表格展示用户名、角色、启用状态、最近登录时间、最近登录 IP、密码更新时间。
- 支持新增、编辑、重置密码、启停。
- 禁止停用或降级最后一个启用管理员时，前端显示明确错误。
- 不展示密码哈希，不回显旧密码。

验收：

- admin 可见用户管理。
- operator 不可见用户管理。
- 用户管理接口返回 `403` 时显示中文无权限提示且不退出登录。

## 7. 权限与错误处理需求

要求：

- `401` 表示登录或会话失效，前端跳回登录页。
- `403` 表示权限不足，前端显示中文错误，不清理 token，不跳回登录页。
- operator 隐藏用户管理、真实 SSH 批量执行、告警规则编辑、告警通知配置和定时任务管理入口。
- 所有高风险操作需要确认弹窗：
  - 删除设备。
  - 删除分组。
  - 删除文件。
  - 真实 SSH 批量执行。
  - 真实 SSH 定时任务执行。
  - 删除或停用用户。
  - 删除告警通知通道或策略。
- 所有连接失败、文件失败、任务失败和通知失败都应尽量局部显示错误。
