# Wave 11 核心加固与运维体验补全需求

## 背景

当前平台已经完成登录认证、设备 CRUD、frps 导入、远程 SSH/VNC WebSocket、文件接口、定时任务接口、批量更新任务、真实 SSH 执行、诊断接口、Postman 集合和 Nginx 单域名部署说明。最新本地提交为 `10db088 Implement Wave 10 batch SSH execution`。

`docs/02-优化建议文档.md` 中包含大量安全、后端质量、前端功能和展示优化建议。本轮不完全照单实现，优先选择能直接提升服务器测试可靠性、降低凭据风险、补齐当前运维闭环的项目。

## 目标

设计下一轮开发计划，建议命名为 Wave 11。Wave 11 应聚焦“核心安全加固 + 常用运维操作补全 + 批量任务可用性提升”，避免一次性展开告警、多用户权限、报表、深色模式、完整 noVNC/xterm 体验等大范围功能。

## 本轮建议纳入范围

### 1. 设备 SSH 凭据加密与使用链路加固

- 新增后端加密服务，使用 `cryptography.Fernet` 或等价对称加密方案。
- 新增配置项，例如 `CREDENTIAL_ENCRYPTION_KEY`，用于加密和解密设备 SSH 密码。
- 设备创建、更新、frps 导入默认凭据写入时保存密文。
- SSH 执行、Web SSH、SFTP 文件后端统一通过设备级凭据解密后连接。
- 保留兼容：如果数据库中已有明文 `ssh_password_encrypted`，本轮可支持读取旧值并在设备更新时迁移，不强制一次性迁移全部历史数据。
- API 响应继续禁止返回明文密码。

### 2. 默认弱配置风险提示和运行时诊断增强

- 对 `JWT_SECRET_KEY=change-me-in-production`、默认管理员密码、默认设备 SSH 密码等弱配置给出明确诊断。
- 在开发环境允许启动，但在诊断接口中标记 `warning`；生产环境是否拒绝启动可作为后续增强，不在本轮强制。
- `GET /api/diagnostics/config` 增加安全摘要字段，但不得泄露密钥、Token、密码、私钥或解密后的凭据。

### 3. 前端设备管理补全

- 补齐设备编辑能力，复用现有创建设备字段，支持更新项目号、位置、分组、标签、SSH 用户、SSH 密码等。
- 补齐设备删除能力，加入确认弹窗，删除后刷新列表。
- 增加单设备状态刷新入口，调用 `GET /api/devices/{id}/status`。
- 对 SSH 密码输入做中文提示：留空表示不修改已有凭据。

### 4. 批量更新任务体验增强

- 创建真实 SSH 任务前增加确认提示，展示执行模式、命令、目标设备数量或筛选条件。
- 任务列表增加取消按钮，调用 `POST /api/update-tasks/{id}/cancel`。
- 任务详情展示继续使用 Wave 10 的设备级结果字段，优化为空值、失败、跳过、退出码非 0 的中文展示。
- 若可控，接入现有 `WS /api/ws/update-tasks/{id}` 快照刷新；若实现成本超过本轮预算，可保留手动刷新。

### 5. Postman 与部署验收更新

- Postman 增加设备编辑、删除、诊断安全摘要、批量任务取消的请求。
- README、API 文档、部署文档补充凭据加密、弱配置诊断和前端操作说明。
- 给出服务器测试建议：先导入 frps 设备，确认凭据，再执行 `hostname` 级别的真实 SSH 任务。

## 本轮不纳入范围

- 完整 Alembic 迁移体系替换。
- 多用户、RBAC、会话管理。
- 告警规则、通知渠道、告警历史。
- xterm.js 完整终端组件和 noVNC 嵌入式客户端。
- ECharts 仪表盘、在线率趋势、报表系统。
- 文件管理完整前端页面。
- 定时任务完整前端页面。
- SSH 主机密钥严格校验和 known_hosts 管理。

这些项目价值明确，但会扩大测试面和交互面，建议放到 Wave 12 之后按模块推进。

## 验收标准

- 后端凭据加密有单元或接口测试覆盖：创建、更新、读取响应不泄密、SSH 服务使用解密后的设备密码。
- 旧数据兼容路径有测试覆盖，不能让现有默认 `123456` 设备在升级后全部不可用。
- 诊断接口只返回安全摘要和 warning，不返回敏感明文。
- 前端设备编辑、删除、状态刷新、批量任务取消和执行确认有 Vitest 覆盖。
- Postman Collection JSON 可解析，新增请求可按登录后 Token 自动执行。
- README、API 文档、部署文档与实际行为一致。
- 最低验证命令：
  - `py -3.12 -m pytest tests --basetemp 'C:\01_work\02_program\远程终端平台\.pytest-tmp'`
  - `npm.cmd test -- --run`
  - `npm.cmd run build`
  - `git diff --check`

## 主要风险

- 加密密钥缺失或轮换策略设计不当会导致历史设备凭据不可用。本轮只做单密钥加解密和兼容读取，不做复杂轮换。
- 前端设备编辑如果把密码字段误回填，会造成明文泄露。本轮要求密码字段永不从 API 回显，只允许用户主动输入新密码。
- 批量任务真实 SSH 已接入，任何 UI 上的误触都可能影响设备。本轮必须增加真实执行确认。
