# Wave 8 开发需求：真实远程连接与文件传输

## 背景

当前平台已经完成后端基础能力、设备管理、分组日志、监控总览、批量更新、定时任务、部署脚本，以及前端真实 API 联调。远程 SSH/VNC 和设备文件接口目前仍停留在“会话描述”和“本地文件模拟”阶段：

- `POST /api/devices/{id}/remote/ssh` 返回 SSH 会话描述，不提供浏览器内真实终端。
- `POST /api/devices/{id}/remote/vnc` 返回 VNC 会话描述，不提供真实 noVNC 代理。
- `/api/devices/{id}/files` 系列接口当前使用服务端本地目录模拟设备侧文件系统。
- 批量更新任务当前仍为模拟执行，真实 SSH 命令执行将留到 Wave 9，Wave 8 只交付可复用的远程连接基础设施。

Wave 8 的目标是把“远程能力骨架”推进为可连接真实设备的第一版能力，并为后续 Wave 9 的真实批量执行复用 SSH/SFTP 基础服务。

## 目标

实现真实远程连接基础能力：

1. 后端提供可测试的 SSH 连接服务，能够通过设备的 frp SSH 端口建立连接、执行基础命令、打开交互式 shell。
2. 后端提供 Web SSH WebSocket，浏览器终端可以与设备 SSH shell 双向通信。
3. 后端提供 SFTP 文件服务适配器，将现有文件接口从本地模拟切换为真实设备 SFTP 操作。
4. 后端提供 VNC WebSocket-to-TCP 代理基础能力，前端可生成 noVNC 可用连接入口。
5. 前端远程连接页面从静态按钮升级为真实会话入口，并展示连接中、已连接、失败、断开等状态。
6. 所有远程操作必须记录操作日志，并对认证、设备不存在、端口不可用、连接超时、认证失败等错误返回可理解的中文提示。

## 范围

### 后端 SSH 服务

- 新增 SSH 适配服务，建议使用 `paramiko`。
- 输入来源使用现有设备字段：`ssh_port`、`ssh_user`、设备标识、平台配置中的 frps/远程主机信息。
- 支持连接超时、认证失败、端口不可达、通道关闭等错误归一化。
- 支持执行轻量探测命令，例如 `echo ok`，供测试和后续健康检查复用。
- 支持打开交互式 shell channel，供 Web SSH WebSocket 转发。
- 不在 Wave 8 中实现批量更新真实命令执行，但服务接口应能被 Wave 9 复用。

### Web SSH

- 新增或补齐 WebSocket 端点，建议形式为 `/api/ws/devices/{device_id}/ssh?token=<access-token>`。
- WebSocket 必须校验 JWT token，拒绝未认证、过期或无效 token。
- 浏览器输入应转发到 SSH channel，设备输出应回传给浏览器。
- 支持前端发送 resize 消息，后端调整伪终端尺寸。
- 支持空闲/连接超时和断开清理，避免残留 SSH session。
- 连接成功、失败和关闭均记录操作日志。

### SFTP 文件传输

- 保持现有 REST 接口路径不变：
  - `GET /api/devices/{id}/files`
  - `POST /api/devices/{id}/files/upload`
  - `GET /api/devices/{id}/files/download`
  - `DELETE /api/devices/{id}/files`
- 将实现从本地存储模拟替换为真实 SFTP 适配。
- 文件列表返回目录项名称、类型、大小、修改时间等基础信息。
- 上传接口继续支持文本/内容上传；如当前 schema 不支持二进制文件，Wave 8 只要求文本和小文件内容路径可用，二进制 multipart 可放到后续扩展。
- 下载接口应从设备 SFTP 读取并流式返回。
- 删除接口应只允许删除文件，不允许越权路径和危险路径。
- 路径安全规则必须保留：禁止空路径、相对逃逸、根路径删除、路径穿越。

### VNC 代理与 noVNC 入口

- 新增后端 WebSocket-to-TCP relay，建议形式为 `/api/ws/devices/{device_id}/vnc?token=<access-token>`。
- 后端连接目标为设备 `vnc_port` 对应的 frp TCP 端口。
- WebSocket 与 TCP socket 双向转发二进制数据。
- 前端提供 noVNC 入口 URL 或嵌入式 noVNC 容器。
- Wave 8 的 VNC 验收目标是代理链路可建立、连接失败可提示、鉴权可拒绝；剪贴板同步、画质调节、多显示器等高级能力不纳入本波。

### 前端远程页面

- 远程连接页面展示设备名称、序列号、在线状态、SSH 端口、VNC 端口。
- SSH 按钮打开真实终端视图，建议集成 `xterm.js`。
- VNC 按钮打开 noVNC 视图或新窗口。
- 展示会话状态：准备连接、连接中、已连接、失败、已断开。
- 用户可主动断开 SSH/VNC 会话。
- 后端返回错误时显示中文错误，不应导致退出登录。

### 操作日志

以下行为必须记录操作日志：

- SSH 连接发起、连接成功、连接失败、主动断开。
- VNC 连接发起、连接成功、连接失败、主动断开。
- SFTP 文件列表、上传、下载、删除。
- 失败日志应包含目标设备和归一化错误摘要，但不得写入明文密码、私钥、token 或文件内容。

## 非目标

- 不实现 Wave 9 的真实批量更新任务执行。
- 不要求实现 SSH 凭据加密存储大改；若当前设备模型缺少密码/私钥字段，Wave 8 可先支持密钥/密码来自安全配置或测试注入，并冻结后续凭据模型升级需求。
- 不实现生产级 noVNC 全功能体验，例如剪贴板同步、动态画质、录屏、多用户协同。
- 不实现大文件断点续传。
- 不引入边缘端 Agent。
- 不改变现有 REST API 路径，除非必须新增 WebSocket 端点。

## 验收标准

### 自动化验收

- 后端新增 SSH 服务单元测试，覆盖连接成功、认证失败、端口不可达、执行命令成功、通道关闭清理。
- 后端新增 WebSocket SSH 测试，覆盖 token 拒绝、设备不存在、连接失败、正常双向转发的可模拟路径。
- 后端新增 SFTP 文件接口测试，覆盖列表、上传、下载、删除、路径穿越拒绝。
- 后端新增 VNC relay 测试，覆盖 token 拒绝、TCP 连接失败、二进制转发的可模拟路径。
- 前端新增远程页面测试，覆盖 SSH/VNC 按钮触发、连接状态展示、失败提示、断开操作。
- 全量后端测试通过。
- 全量前端测试通过。
- 前端构建通过。

### 手工联调验收

- 使用一台可通过 frp SSH 端口访问的测试设备，浏览器能打开 SSH 终端并执行 `echo ok`。
- 对同一测试设备执行文件列表、上传小文本文件、下载校验内容、删除文件。
- 如测试设备具备 VNC 服务，浏览器能通过 noVNC 入口进入连接流程；若无真实 VNC 服务，至少验证代理连接失败路径可读、不会卡死或退出登录。
- 后端日志和操作日志中能看到 SSH/VNC/SFTP 的成功与失败记录。

## 测试设备与配置要求

执行 Wave 8 前需要明确以下信息：

- frps 或平台后端访问设备 SSH/VNC 端口时使用的主机地址：默认可假设为 `127.0.0.1` 或配置项。
- 测试设备 SSH 凭据来源：密码、私钥，或本地测试 mock。
- VNC 是否有可用测试设备和认证密码。
- 是否允许在测试设备 `/tmp` 或指定目录创建、读取和删除验收文件。

若这些信息暂时无法提供，Wave 8 仍可先实现可注入适配器和 mock 集成测试，但最终完成状态必须标记为“需要真实设备手工验收”。

## 风险与约束

- WebSocket 到 SSH/VNC 的双向转发涉及线程、异步任务和资源清理，必须避免连接泄漏。
- Paramiko 是阻塞库，FastAPI WebSocket 端点中需要明确线程/任务边界，不能阻塞事件循环。
- VNC 是二进制流，前端和后端都必须按二进制 WebSocket 处理。
- 凭据、token、私钥、文件内容不得写入日志。
- 当前前端是单页集中式结构，Wave 8 如引入 xterm/noVNC，应控制改动范围，避免顺手重构整个前端。
