# Wave 13-17 后续优化路线图

## 当前基线

截至 Wave 12，平台已经具备以下核心能力：

- 登录、JWT 鉴权、修改管理员密码。
- 设备 CRUD、frps Dashboard 导入、设备级 SSH 凭据、凭据加密配置和兼容旧明文读取。
- 远程 SSH/VNC WebSocket 基础代理。
- 文件接口、定时任务接口、批量更新任务模型和真实 SSH 批量命令执行。
- 分组 CRUD、设备按分组筛选、操作日志筛选分页和 CSV 导出。
- 诊断接口、诊断页面、Postman Collection、Nginx 单域名部署说明。

原 `docs/02-优化建议文档.md` 中已经落地或部分落地的内容包括：凭据加密、设备级 SSH 凭据、默认弱配置诊断、设备编辑/删除/状态刷新、改密、分组增删改、日志导出、同步配置和系统诊断页。后续优化应聚焦剩余高价值项，避免一次性扩展到多用户、告警、报表和复杂 UI 重构。

## Wave 13：监控指标与仪表盘可观测性

### 目标

把当前“资源快照为 0 或缺少趋势”的问题收敛掉，让管理员登录后能看到设备运行状态、资源指标和异常设备摘要。

### 范围

- 后端补齐或复核设备指标接口：
  - `GET /api/devices/{id}/metrics`
  - 指标字段至少覆盖 CPU、内存、磁盘、采集时间。
  - 无真实采集数据时返回空列表，不伪造成功指标。
- 前端仪表盘接入真实指标：
  - 设备资源卡片显示真实 CPU/内存/磁盘。
  - 无数据时显示“暂无指标”，不显示 0% 误导。
  - 增加异常设备区：离线、未知、指标过高。
- 增加基础图表：
  - 设备状态分布。
  - 在线/离线趋势或最近指标趋势。
  - 可使用 ECharts，但只引入必要图表，避免大范围重构。
- 诊断页增加监控数据可用性摘要：
  - 最近指标时间。
  - 指标表是否为空。

### 验收

- 前端测试覆盖无指标、有指标、异常设备展示。
- 后端测试覆盖指标列表和空数据。
- `npm.cmd test -- --run`、`npm.cmd run build`、后端 pytest 通过。

## Wave 14：远程连接体验产品化

### 目标

把 Wave 8 的 SSH/VNC WebSocket 基础能力提升为可用的远程操作体验，减少“只有会话描述、缺少真实终端 UI”的落差。

### 范围

- SSH 页面集成 xterm.js：
  - 支持终端输出、输入、断开、重连。
  - 支持窗口 resize 上报。
  - 显示连接状态和失败原因。
- VNC 页面集成 noVNC：
  - 使用当前 VNC WebSocket 代理能力。
  - 支持连接、断开、全屏入口。
  - 不做复杂剪贴板、文件拖拽等增强。
- 增加远程会话审计：
  - 连接成功、失败、断开记录操作日志。
  - 不记录明文密码、Token、私钥内容。
- 前端远程页改为左右布局：
  - 左侧设备列表。
  - 右侧 SSH 终端或 VNC 画面。

### 验收

- Vitest 覆盖 SSH 连接状态、输入发送、断开。
- 浏览器手工验收至少覆盖一个可达 SSH 设备。
- 构建通过，远程失败时页面不闪退。

## Wave 15：文件管理与定时任务前端闭环

### 目标

后端已有文件接口和定时任务接口，但前端缺少完整入口。本轮补齐两个“已有 API 但无法在 UI 使用”的模块。

### 范围

- 文件管理页或设备详情文件面板：
  - 浏览目录。
  - 上传文件。
  - 下载文件。
  - 删除文件。
  - 支持本地后端和 SFTP 后端的错误提示。
- 定时任务管理页：
  - 任务列表。
  - 创建、编辑、删除。
  - 启用/停用。
  - 立即执行。
  - 查看任务执行日志。
- 前端 API 封装补齐：
  - 文件接口。
  - scheduled-tasks 接口。
- Postman 增加文件和定时任务验收请求。

### 验收

- Vitest 覆盖文件列表、上传、删除、定时任务创建和启停。
- 后端现有文件/定时任务测试继续通过。
- 文档补充前端操作路径和 SFTP 部署注意事项。

## Wave 16：批量任务增强与设备选择器

### 目标

提升真实 SSH 批量任务的安全性和可控性，让管理员在执行前能明确知道目标设备、命令内容和风险。

### 范围

- 设备选择器：
  - 支持按项目、分组、标签筛选。
  - 支持手动勾选设备。
  - 创建任务前展示命中设备数量和设备清单。
- 任务模板：
  - 保存常用命令模板。
  - 模板只保存命令和说明，不保存设备凭据。
- 批量任务详情增强：
  - 行展开展示每台设备 stdout/stderr 摘要、退出码、错误原因。
  - 支持重新执行失败设备。
  - 支持导出任务结果 CSV。
- WebSocket 实时进度：
  - 接入 `WS /api/ws/update-tasks/{id}`。
  - 前端显示实时状态，不依赖手动刷新。

### 验收

- 创建真实 SSH 任务前必须出现二次确认和目标设备清单。
- Vitest 覆盖设备选择、模板套用、任务结果展开、取消任务。
- 后端测试覆盖任务结果导出和失败设备重试。

## Wave 17：后端治理、安全边界与长期维护

### 目标

在核心业务功能可用后，降低长期维护风险，解决优化建议中“代码重复、Schema 约束弱、迁移粗糙、测试重复”的问题。

### 范围

- Schema 枚举约束：
  - `status`、`ssh_auth_type`、`task_type`、`execution_mode` 等字段改为 `Literal` 或 Enum。
  - 保持向后兼容，对非法值返回 422。
- 测试辅助收敛：
  - 提取重复 `_auth_headers` 为 pytest fixture。
  - 统一测试设备、分组、任务 factory。
- Router 依赖收敛：
  - 提取数据库 session 依赖。
  - 降低每个 router 中重复 `get_app_settings + session_scope` 的样板代码。
- 数据库迁移治理：
  - 引入 Alembic 或最小迁移层。
  - 现有 SQLite 自动补列逻辑保留兼容，但新增结构变更必须有版本化迁移。
- SSH 主机密钥策略：
  - 增加 known_hosts 配置能力。
  - 默认开发环境可兼容，生产环境给出明确诊断 warning。
- Token 自动刷新：
  - axios 401 时尝试 refresh token。
  - refresh 失败再清理登录态。

### 验收

- 后端全量 pytest 通过。
- 前端登录态过期测试覆盖 401 refresh 和 refresh 失败。
- 诊断页能提示迁移状态、known_hosts 和弱配置状态。
- 文档明确生产环境的 JWT、凭据加密、known_hosts 和数据库迁移建议。

## 暂缓项

以下内容价值存在，但不建议在 Wave 13-17 期间优先展开：

- 多用户 RBAC。
- 告警规则、Webhook、邮件通知。
- 完整报表中心。
- 设备生命周期资产管理。
- 深色模式、全局快捷键、主题系统。
- 大规模 UI 重构或引入 Pinia/Vue Router 全量改造。

这些能力应在远程体验、文件/定时任务、批量任务和后端治理稳定后，再按真实使用反馈拆分。
