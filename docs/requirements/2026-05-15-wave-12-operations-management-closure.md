# Wave 12 运维管理闭环补全需求

## 背景

平台已完成 Wave 11：设备 SSH 凭据加密、弱配置诊断、设备编辑/删除/状态刷新、真实 SSH 批量任务确认与取消、Postman 和部署文档同步。当前后端已经暴露多组实用接口，但前端仍有部分关键运维入口缺失或体验较弱。

`docs/02-优化建议文档.md` 中仍有大量建议。本轮不展开大型新模块，优先选择“后端已有能力、前端缺入口、服务器测试会频繁用到”的功能，补齐日常运维闭环。

## 目标

Wave 12 聚焦“账号、分组、日志、设备配置同步和诊断页”五类轻量但高频的管理能力，让平台在服务器测试和日常运维中更完整：

- 管理员可以在前端修改密码。
- 分组可以在前端创建、编辑、删除，并与设备列表联动。
- 操作日志可以筛选、分页和导出 CSV。
- 设备可以从前端触发 `frpc` 配置同步并查看生成内容。
- 诊断配置可以在前端查看，用于部署后快速排查安全配置、网关、数据库和文件后端状态。

## 本轮纳入范围

### 1. 管理员密码修改

- 前端顶栏增加用户入口，展示当前管理员身份。
- 增加“修改密码”对话框。
- 调用 `PUT /api/auth/password`。
- 表单字段：
  - 当前密码
  - 新密码
  - 确认新密码
- 前端校验：
  - 当前密码必填。
  - 新密码与确认密码一致。
  - 新密码长度满足后端要求。
- 成功后提示重新登录，并清除本地 token。
- 失败时显示中文错误提示，不重复弹出同一错误。

### 2. 分组管理 CRUD 与设备联动

- 分组页增加“新建分组”按钮。
- 分组卡片增加编辑和删除入口。
- 调用后端已有接口：
  - `POST /api/groups`
  - `PUT /api/groups/{id}`
  - `DELETE /api/groups/{id}`
- 分组表单字段：
  - 分组名称
  - 父分组 ID 或父分组选择
  - 描述
- 删除前确认，避免误删。
- 分组卡片显示设备数量，并可点击跳转到设备管理页按分组筛选。
- 设备管理页增加分组筛选入口，至少支持按分组点击过滤本地列表。

### 3. 操作日志筛选、分页与 CSV 导出

- 操作日志页增加筛选控件：
  - 操作类型
  - 状态
  - 目标类型
- 接入后端日志查询参数：
  - `action`
  - `status`
  - `target_type`
  - `offset`
  - `limit`
- 增加分页组件。
- “导出 CSV”按钮调用 `GET /api/logs/export`。
- 下载文件名建议为 `operation_logs.csv`。
- 前端导出时继续带 Bearer Token。
- 若后端 CSV 导出仍缺少 `Content-Disposition` 或 CSV 注入防护，本轮应补齐后端安全处理。

### 4. 设备 frpc 配置同步入口

- 设备表格或设备编辑面板增加“同步配置”按钮。
- 调用 `POST /api/devices/{id}/sync-config`。
- 成功后展示生成的 frpc 配置内容，可复制。
- 操作日志刷新，便于确认同步动作已记录。
- 当前后端仍只生成配置内容，不真实写入设备；前端文案必须明确“生成配置”，不能暗示已经下发到设备。

### 5. 前端诊断配置页

- 新增导航入口“系统诊断”或在日志/设置区域增加诊断面板。
- 调用 `GET /api/diagnostics/config`。
- 展示内容：
  - 服务名称、版本、API 前缀
  - 数据库摘要
  - 文件后端
  - SSH/VNC 网关主机
  - SSH/VNC 超时
  - 默认设备 SSH 用户
  - Wave 11 的 `security` 摘要和 warning 列表
- 敏感信息不得在前端显示。
- warning 使用中文醒目样式，但不要阻止其他功能使用。

## 可选小优化

如实现成本较低，可顺手纳入：

- 设备标签点击后按标签过滤设备列表。
- 设备列表空状态提示“创建第一台设备”。
- 批量更新进度从纯数字改为进度条。

这些是可选项，不应影响主目标交付。

## 本轮不纳入范围

- 完整定时任务前端页面。
- 完整文件管理前端页面。
- ECharts 仪表盘、在线率趋势、报表系统。
- 多用户、角色权限、会话管理。
- 告警规则、告警通知、告警历史。
- xterm.js 终端组件和 noVNC 嵌入。
- Alembic 全量迁移替换。
- SSH known_hosts 严格主机密钥管理。

上述项目继续保留为后续 Wave。

## 验收标准

### 后端

- 如果修改日志导出：
  - CSV 响应包含合理的下载头。
  - CSV 字段具备基础注入防护。
  - 现有日志导出测试通过，并新增覆盖危险前缀字段的测试。
- 其他后端改动不得破坏现有 48 个测试。

### 前端

- 管理员修改密码对话框有 Vitest 覆盖：
  - 校验失败不调用 API。
  - 成功后清 token 并回到登录页。
- 分组 CRUD 有 Vitest 覆盖：
  - 创建、编辑、删除调用正确 API。
  - 删除前确认。
  - 点击分组能切到设备列表并过滤。
- 日志筛选、分页、导出有 Vitest 覆盖：
  - 筛选条件传给 API。
  - 分页改变 offset/limit。
  - 导出按钮调用下载逻辑。
- 设备同步配置有 Vitest 覆盖：
  - 调用同步接口。
  - 展示返回配置内容。
- 系统诊断页有 Vitest 覆盖：
  - 调用诊断接口。
  - 展示 security warning。
  - 不展示密钥、密码、Token 等敏感字段。

### 文档与 Postman

- README 更新 Wave 12 能力说明。
- API 文档补充日志导出安全行为、诊断页使用说明。
- 部署文档补充部署后建议检查系统诊断页。
- Postman Collection 增加：
  - 修改密码
  - 分组创建/编辑/删除
  - 日志筛选和导出
  - 设备同步配置
  - 诊断配置
- Postman JSON 可解析。

## 风险与约束

- 修改密码会影响当前登录态，前端必须在成功后清 token 并提示重新登录。
- 分组删除可能影响设备显示，若后端当前删除行为会保留设备 `group_id` 或产生约束问题，执行计划阶段需先核对并按现有后端语义处理。
- 日志导出涉及浏览器下载逻辑，测试应 mock `Blob`、`URL.createObjectURL` 和临时链接点击。
- 诊断页只能展示摘要，不能为了“排查方便”显示真实密钥、密码、Token 或私钥路径内容。
- 本轮仍保持前端中文显示。
